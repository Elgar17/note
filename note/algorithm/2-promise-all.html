# promise

缺点：

  1. 无法取消 Promise,一旦新建它就会立即执行，无法中途取消。
  2. 如果不设置回调函数，promise 内部抛出的错误，不会反应到外部。
  3. 当处于 pending 状态时，无法得知目前进展到哪一个阶段（刚刚开始还是即将完成）。

优点：

  1. 解决回调地狱（Callback Hell）问题
  2. 代码更扁平，可读性更高
  3. 更好地进行错误捕获

# 描述Promise all

实现一个 Promise-all 函数，传入多个 Promise 需要返回一个新的 Promise，并且返回传入 Promise 的所有结果。

# 实现Promise all

function PromiseAll(...arg){
    var resultArr = [];
    return new Promise((resolve,reject)=>{
        let i = 0;
        function next(){
            arg[i].then(data=>{
                resultArr.push(data);
                i++;
                if(i == arg.length){
                    resolve(resultArr)
                }else{
                    next();
                }
            }).catch(err=>{
                reject(err)
            })
        }
        next();
    })

}

# 描述Promise race

实现一个 Promise.race函数，传入多个 Promise 需要返回一个新的 Promise，并且返回传入 Promise 中最先返回的 promise 中的结果。

# 实现 promise.race

function PromiseRace(...argArr) {
    return new Promise((resove, reject) => {
        for (let i = 0; i < argArr.length; i++) {
            argArr[i].then(data => {
                resove(data)
            })
        }

    })
}

// 简化版本
function PromiseRace(...argArr) {
    return new Promise((resove, reject) => {
        argArr.forEach(promise=>{
            promise.then(resove,reject)
        })
    })
}
