    # 1、cookie & session

    # 1.1、cookie

    cookie存储在浏览器的键值对（key=value）字符串。

    有 5 kb 的大小限制，跨域不共享。

    当每次发送请求时，将cookie发送给服务器。

    在浏览器查看cookie 浏览器输入以下命令或值在network面板中找到对应的cookie即可。

    document.cookie

    服务端操作 cookie (nodejs操作)

    // 获取
req.headers.cookie

// 操作 cookie
res.setHeader('Set-Cookie',`username=${data.username}; path=/; httpOnly;  expires=${getCookieEexpries()}`)
// path=/： 告诉浏览器cookie的使用范围时 / 路由
// httpOnly： 限制用户（客户端）更改cookie
// expires=${getCookieEexpries()}:设置过期时间

    # 1.2、session

    解决什么问题？

    cookie发送个客户端容易暴露个人信息，所以给客户端的是一个不重复的随机数，在服务端存着（session中）这个随机数对应用户的名字，这样可以验证用户的信息了。

    session存在什么问题？

    问题1：session存放在服务器内存中的，所以访问量过大，session也一起扩大，导致内存暴增。

    问题2：线上环境在多线程 session 无法共享session。

    解决方案：redis

    # 2、session 写入 redis

    redis 是内存数据库

    redis是常用的缓存数据库，数据存储在内存中。

    redis 是单独开辟一个进程，其他进程都可以访问，

    安装：

    下载完成后进入redis目录

    cd D:\Software2\redis


// 启动

    新建一个cmd窗口 之前串口不要关，切换到 redis目录

redis-cli.exe -h 127.0.0.1 -p 6379

// 设置

// 读取

// 获取所有redis

// 删除

    # 2.1、node 连接 redis

    安装

    npm i redis --save

    使用

    const redis = require('redis')

// 创建客户端 启动服务的端口 访问地址 
const redisClient = redis.createClient(6379,'127.0.0.1')

// 发生错误处理
redisClient.on('error',err=>{
    console.log(err)
})

// 使用 第三个参数打印 ok
redisClient.set('myname','Tom',redis.print)
redisClient.get('myname',(err,val)=>{
    if(err){
        console.log(err)
    }
    console.log(val)
    // 退出
	redisClient.quit()
})

    # 3、nginx 反向代理

    # 3.1、nginx 是什么

    nginx是高性能 web 服务器，开源并且免费。

    nginx 有以下用处

    • 静态服务
    • 负载均衡
    • 反向代理

    这里只介绍反向代理。

    前后端分离的项目中，我们需要开启两个服务，所以存在跨域问题

    这里 nginx 做的就是，我们启动两个服务，比如我们项目静态页面在 5500 端口上运行的，后端运行在 8088 端口上，当我们访问想获取后端数据是存在跨域问题。

    我们安装 nginx 后把 nginx 启动在默认的80端口上，nginx 的作用是 当用户访问路径 以 / 开头时，代理到5500 端口上，当用户访问 /api/开头是代理到 8088 端口上。

    这样可以解决跨域问题。

    # 3.2、安装 & 配置

    打开下载的目录找到 nginx/fon/nginx.conf 文件

    // 需要注释
#    location / {
#        root   html;
#        index  index.html index.htm;
#    }
#}

    自己添加配置nginx/fon/nginx.conf

    
# / 开头的请求代理到http://localhost:5500;
location / {
	proxy_pass: http://localhost:5500;
}
 
# /api/ 开头的请求代理到http://localhost:8088;
location /api/ {
	proxy_pass: http://localhost:8088;
	proxy_set_header Host $host;
}

    在根目录打开 cmd 输入 nginx -t 测试，输出以下内容说明安装成功。

    D:\Software2\nginx>nginx -t
nginx: the configuration file D:\Software2\nginx/conf/nginx.conf syntax is ok
nginx: configuration file D:\Software2\nginx/conf/nginx.conf test is successful

    启动：

    nginx
