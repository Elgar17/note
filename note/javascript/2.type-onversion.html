# 类型转换

# 1、前言

JavaScript是个很灵活的语言，之所以很灵活是因为它有着很强大的类型转换，你是不是很掌握了类型转换？来几道题试一下自己的水平😏。

typeof(a)			// 未定义a
typeof(typeof(a))	
typeof(undefined)	
typeof(NaN)
typeof(null)
var a = "12ab"
typeof(a)
typeof(+a)
typeof(!!a)
typeof(a = "")
1 == "1"
NaN == NaN
NaN === undefined
11 + "11"
1 === "1"
parseInt("123abc")
var num = 123.456
num.toFixd(2)

不太会？不慌现在看一下还来得及。

JavaScript 类型转换分为类型转换和隐式类型转换。

# 2、显示类型

  • Number(mix)：转化成数字
  • parseInt(str,radix)：转化成整数
  • parseFloat(str)：转化成浮点数
  • toString(radix)：转化成字符串，null和undefined没有toString
  • string()：写什么都转化成字符串
  • Boolean()：转化成布尔值
Number("123")  // 123
Number(true)	// 1
Number(false)   // 0
Number(null)	// 0 ⭐
Number(undefined)	// NaN⭐
Number("a")			// NaN

parseInt("123.9")		// 123
parseInt(true)			// NaN🌟
parseInt(10,16)			// 16 进值
parseInt("100px")		// 100

parseFloat("33.33px")	//33.33

String(true)			// "true"
Strig(1)				// "1"

var num = 10
str.toString()			// 返回"10"
str.toString(8)			// 返回 "12"转化成8进值
typeof num				// "number"

题：将二进制数 10101010 转化成16进值。

var a = 10101010		
var b = parseInt(a,2)	// 170
b.toString(16)        	// "aa"

# 3、隐式类型转换

隐式类型转换发生在进行这些运算的时候。

  • isNaN(x)：里面的参数放在Number(x)中与NaN比较并且返回结果。

  • ++/-- +/-：先隐式进行Number()运算

  • +：符号一边有字符串,+号会转化成字符串⭐

  • - * / %：先两边的变量都转化成数字

  • && || ！：非将变量转成布尔值，然后取反。

  • < > >= <= ：数字和字符串比较，转化成数字。两边都字符串比较ASCII码大小。

  • == / !=与=== / !==：== / !=存在隐式类型转换，=== / !==不存在类型转换

isNaN(NaN)		// true
isNaN("a")		// true  -> Number("a") -> NaN === NaN
isNaN(null)		// false

var i = "0";
i++			// 1
typeof(i)	//"number"

var str = "abc"
str++		// NaN  Number("abc") -> NaN
typeof(str)	//	"number" 

// 一元运算父转换成 Number 类型
str = +"a"	//	NaN 		
typeof(str)	//	"number" 
str = -"a"	//	NaN 
typeof(str)	//	"number" 

// + 号
"a" + 1		// "a1"	字符串
1 + "A"		// "1A" 字符串

// - * / % 
1 * "2"		// 2 number 类型
"a" * 1		// NaN Number("a") -> NaN * a -> NaN

1 > "a"		// false
1 < "a"		// false
"b" > "a"	// true  98 > 97
true > false //true

123 == "123"		// true
"123" === "123"		// false

NaN > 0   // false
NaN < 0   // false  记住这些以下几个也可以推出来
undefined > 0 // Number(undefined) -> NaN < 0 -> false

知道这些以后，都有自己的转换规则，但是还需要记住一些特殊的,没有规则可循。:undefined == null值为true， NaN == NaN返回false。

只有一种情况未定义的变量不报错：

typeof(a)			// undefined 这是字符串
typeof(typeof(a))	// string

# 4、答案

typeof(a)			// "undefined"
typeof(typeof(a))	// "string"
typeof(undefined)       // "undefined"
typeof(NaN)             // "number"⭐
typeof(null)            // "object"⭐
var a = "12ab"		
typeof(a)			// "string"
typeof(+a)			// "number"
typeof(!!a)			// "boolean"
typeof(a = "")		// "string"
1 == "1"			// true
NaN == NaN			// false
NaN === undefined	// false
11 + "11"			// "1111"
1 === "1"			// false
parseInt("123abc")	// 123
var num = 123.456	
num.toFixed(2)		// "123.45"
